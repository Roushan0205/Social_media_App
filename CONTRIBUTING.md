Code of Conduct
By participating in this project, you agree to maintain a respectful and inclusive environment for everyone. Please be kind and courteous in all interactions.
How Can I Contribute?
There are many ways to contribute to this project:

🐛 Report bugs - Help us identify issues
💡 Suggest features - Share your ideas for improvements
📝 Improve documentation - Help make our docs clearer
💻 Write code - Implement new features or fix bugs
🎨 Design improvements - Enhance the UI/UX
✅ Review pull requests - Help review others' contributions

Getting Started
Prerequisites
Make sure you have the following installed:

Git
[List any required software, languages, or tools specific to your project]

Fork and Clone

Fork the repository by clicking the "Fork" button at the top right of the repository page
Clone your fork:

bash   git clone https://github.com/YOUR-USERNAME/Social_media_App.git
   cd Social_media_App

Add the upstream remote:

bash   git remote add upstream https://github.com/Roushan0205/Social_media_App.git

Create a new branch for your contribution:

bash   git checkout -b feature/your-feature-name
Development Workflow

Keep your fork updated:

bash   git fetch upstream
   git merge upstream/main

Make your changes in your feature branch
Test your changes thoroughly before committing
Commit your changes (see commit message guidelines below)
Push to your fork:

bash   git push origin feature/your-feature-name

Open a Pull Request from your fork to the main repository

Coding Standards
To maintain code quality and consistency:

✅ Write clean, readable, and maintainable code
✅ Follow the existing code style and structure
✅ Comment complex logic and functions
✅ Use meaningful variable and function names
✅ Keep functions small and focused on a single task
✅ Remove unused code and console logs before committing
✅ Test your code before submitting

Commit Message Guidelines
We follow conventional commit messages for clarity:
Format
<type>(<scope>): <subject>

<body>

<footer>
Types

feat: A new feature
fix: A bug fix
docs: Documentation changes
style: Code style changes (formatting, missing semicolons, etc.)
refactor: Code refactoring without changing functionality
test: Adding or updating tests
chore: Maintenance tasks, dependencies, etc.

Examples
bashfeat(auth): add Google OAuth login
fix(profile): resolve avatar upload issue
docs(readme): update installation instructions
style(css): format styling with prettier
Pull Request Process

Ensure your code follows the coding standards
Update documentation if you're adding new features
Write a clear PR description:

What changes did you make?
Why did you make these changes?
What issue does this PR address? (use Closes #issue-number)


Link related issues in your PR description
Wait for review - A maintainer will review your PR

Address any requested changes
Be patient and respectful during the review process


Once approved, your PR will be merged! 🎉

Reporting Bugs
Found a bug? Help us fix it!
Before submitting a bug report:

Check if the bug has already been reported in Issues
Try to reproduce the bug to confirm it exists

How to submit a bug report:

Go to the Issues page
Click "New Issue"
Use a clear and descriptive title
Provide detailed information:

Steps to reproduce the bug
Expected behavior
Actual behavior
Screenshots (if applicable)
Environment details (OS, browser, version, etc.)



Suggesting Features
Have an idea for a new feature?
Before suggesting:

Check if the feature has already been suggested in Issues
Consider if it aligns with the project's goals

How to suggest a feature:

Go to the Issues page
Click "New Issue"
Use a clear and descriptive title
Explain:

What problem does this feature solve?
How should it work?
Why is this feature valuable?
Any examples or mockups (if applicable)
